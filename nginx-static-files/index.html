Welcome to EDN WebApp

What is this?

Introduction

Welcome to the landing page of the EDDN Elastic WebApp project! This project aims to provide a comprehensive learning experience for working with real-time data streams using Elasticsearch, Node.js, WebSockets, and the EDDN feed. Through this project, you'll gain hands-on experience and in-depth knowledge of these technologies while building dynamic web applications.

Components of the Project

Elasticsearch

Elasticsearch is a powerful search and analytics engine that acts as the backbone of our real-time data processing. It enables efficient storage, retrieval, and indexing of data, allowing us to build robust search and visualization features.

Learn more about Elasticsearch.

Node.js

Node.js is a server-side runtime environment that allows us to build scalable and event-driven applications. With Node.js, we can handle real-time data streams efficiently and leverage the power of JavaScript in server-side development.

Learn more about Node.js.

WebSockets

WebSockets enable real-time bidirectional communication between the client and server. By establishing a persistent connection, we can achieve instant data updates and create interactive web applications that respond in real-time.

Learn more about WebSockets.

EDDN Feed

The EDDN (Elite Dangerous Data Network) feed is a real-time data stream that provides a wealth of information about various events and market prices. By integrating the EDDN feed into our applications, we can create dynamic visualizations and keep users updated with the latest data.

Learn more about the EDDN project.

Metricbeat

Metricbeat is a lightweight shipper that collects and sends system-level metrics and statistics to Elasticsearch. By utilizing Metricbeat, we can monitor CPU usage, memory utilization, network traffic, and other system-level metrics to gain insights into the performance of our infrastructure.

Learn more about Metricbeat.

Filebeat

Filebeat is a lightweight shipper that monitors log files and sends log events to Elasticsearch. We use Filebeat to collect and analyze log data, allowing us to gain insights into the behavior and health of our system. Additionally, Filebeat can be easily configured to handle different types of log files and ship them to Elasticsearch for further processing.

Learn more about Filebeat.

Custom EDDN Parser

In addition to the core components, we have developed a custom parser specifically designed to handle and process the EDDN (Elite Dangerous Data Network) feed. This parser allows us to extract valuable data from the EDDN feed and integrate it seamlessly into our real-time data processing pipeline.
